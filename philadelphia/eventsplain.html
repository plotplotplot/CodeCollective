**Talk 1 — Lakebase: Fully Managed Postgres for the Lakehouse**
*Ayman El-Ghazali · Senior Solutions Architect, Databricks*
Lakebase is a fully managed, Postgres-compatible OLTP database engine for Databricks that seamlessly supports reverse ETL patterns — syncing curated Lakehouse analytics directly into operational Postgres tables for instant use in applications, AI/BI dashboards, and feature stores.
This architecture lets developers build data-driven apps with millisecond latency and transactional reliability, while also activating ML-powered insights in real time — all without custom pipelines or complicated ETL development.
In this session, Ayman will walk through key capabilities, example use cases, and how Lakebase simplifies infrastructure while unlocking new possibilities for AI and analytics.
**Talk 2 — From Legacy Lab Sprawl to Algorithm: Building an AI-Ready Lakehouse for Precision Diagnostics**
*Jim Havrilla, PhD*
A precision diagnostics lab — specializing in rapid pooled antibiotic susceptibility testing for complicated UTIs — inherited a decade of fragmented data: aging SQL instances, LIS/RCM CSVs, and R&D spreadsheets never meant for analytics.
This talk walks through:

* The decision to build a Medallion-architecture lakehouse on Databricks with Unity Catalog
* The Lakeflow Connect integration that unified Salesforce with lab data
* The patent-pending ML model now predicting antibiotic susceptibility from PCR and demographic inputs ahead of phenotypic results

From PRD to Prototype: Hands-On Vibe Coding for the Product Team

**The Problem We're Solving**
Based on Product Tank’s recent survey, most PMs are stuck at "ChatGPT writes my PRD" - only a handful are actually building anything with AI. There's a massive gap between using AI as a writing assistant and using it to create working prototypes.

This workshop bridges that gap. Attendees will go from a Product Requirements Document to a live prototype they built themselves - in less than 90 minutes.

**What Attendees Will Build**
Attendees will pick a project that adds value to their day-to-day. Below is a list of starter options, but feel free to bring your own idea.

1. **Product landing page** — explain a feature or concept, collect interest (featured/recommended)
2. **Interactive prototype** — a clickable multi-step flow or onboarding sequence
3. **User feedback collector** — submit feedback, tag by theme, see it displayed
4. **Feature voting board** — ranked list of ideas your team can interact with

**Every attendee leaves with:**

* A PRD they’ve developed themselves
* A live website URL they can share
* The actual code and files they can modify later
* Proof they can build working prototypes, not just documents

**Why This Approach Works**

**We're teaching real vibe coding** \- the same professional workflow using Claude Code\- not just drag\-and\-drop tools like Webflow or Wix\. This means:

* **Unlimited creative freedom** \- Build anything you can imagine\, not just pre\-built templates
* **Transferable skills** \- Learn professional skills that work for any project

Key insight: **PMs who can prototype their own ideas move faster and communicate more effectively with engineering teams**. This isn't about becoming a developer - it's about removing the dependency on scarce engineering resources for early validation.

**What Attendees Need**

* Laptop (Mac or Windows)
* Wifi/internet connection
* Claude code for desktop
* No coding experience required.
